## Architecture

Static Astro 6 + TypeScript + Tailwind 4 site. Content lives as MDX in
`src/content/writing/`, validated by Astro Content Collections (Zod schema
in `src/content.config.ts`). Hosted on Cloudflare Pages (DNS also on
Cloudflare). Privacy-friendly analytics via GoatCounter (no cookies/PII,
loaded in `BaseLayout.astro`); no comments â€” engagement routes to email / X
via the footer.

### Content pipeline

- Tags are free-form on each post.
- `src/lib/buckets.ts` resolves tags to a fixed set of 5 display buckets
  (Leadership, Engineering, AI, Process, Notes). Each bucket has a distinct
  color treatment in `PostCard.astro` and `PostHeader.astro`. Both the site
  and the drafting agent read this module â€” it's the single source of truth.
- Post URLs are `/blog/<filename-without-extension>`. Nested folders become
  nested routes.
- Each post's frontmatter `hero` carries the cover (`hero.image` + `alt`), with
  an optional `hero.style` (`line-art` | `conceptual`) overriding the auto-picked
  AI cover look â€” see "AI cover generation" below.

### Design system

- Palette tokens (`ink`, `ink-soft`, `ochre`, `terracotta`, `paper`, `paper-soft`)
  are defined as Tailwind 4 `@theme` tokens in `src/styles/global.css`.
  Dark mode inverts via a `.dark` class on `<html>`; default follows
  `prefers-color-scheme`.
- Typography: Fraunces (display, italic-serif), Inter (sans body),
  JetBrains Mono (eyebrows, code).

### Open Graph images

Build-time Open Graph PNGs (1200Ã—630), one per non-draft post, generated by
Satori + `@resvg/resvg-js` (no headless browser): render core in `src/lib/og/`,
emitted by the static endpoint `src/pages/og/[...slug].png.ts` to
`dist/og/<slug>.png`. Layout is **hybrid** (the post's cover under a dark scrim +
title overlay) when `hero.image` exists, and a **branded fallback** (ink panel +
line-art motif + title) otherwise. `BaseLayout.astro` emits `og:image` +
`twitter:card`; post pages pass the `/og/<slug>.png` URL. Gotcha: Satori needs
TTF/OTF, so static fonts are vendored in `src/assets/og/fonts/` â€” the
`@fontsource-variable/*` packages ship WOFF2, which Satori can't read.

### AI cover generation

`scripts/cover/` generates an on-brand cover for a post via Azure OpenAI
`gpt-image-1`, guarded by a `gpt-4o-mini` vision check that rejects text-leaking
images (up to 3 generation attempts, then a deterministic branded fallback that
reuses the OG renderer). It writes `public/static/images/blog/<slug>/cover.png` and sets the
post's `hero` frontmatter. Two interchangeable house styles â€” `line-art` and
`conceptual` â€” are auto-picked by bucket (engineering/ai/process â†’ line-art;
leadership/notes â†’ conceptual) and overridable via `hero.style`. Three entry
points share the orchestrator (`scripts/cover/generate-cover.ts`): the `gen:cover`
CLI, the `gen:cover:all` backfill, and a **non-fatal** hook in the Plan B agent's
draft step. Azure creds are read lazily (`scripts/cover/config.ts`):
`AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_KEY`, `AZURE_OPENAI_IMAGE_DEPLOYMENT`
(default `gpt-image-1`), `AZURE_OPENAI_VISION_DEPLOYMENT` (default `gpt-4o-mini`)
â€” all optional; the site build never calls Azure.

### Custom MDX components

`src/components/mdx/` holds `Callout`, `Youtube`, `Image`, `Toc`. They're
injected into MDX via `<Content components={...} />` in
`src/pages/blog/[...slug].astro`.

### Conventions

- Prettier with `prettier-plugin-astro`. Run `npx prettier --write .` to format.
- Strict TypeScript (`tsconfig.json` extends `astro/tsconfigs/strict`).
- Don't use the Tailwind 3 `tailwind.config.js` pattern â€” tokens live in
  CSS via `@theme`.

## Writing posts

Prose for public-facing pages (blog posts, about, projects) follows
`docs/EDITORIAL.md`. When drafting or editing post content, treat that
file as required reading â€” it documents voice, AI-tell phrases to
avoid, em-dash cap, length-per-bucket, and the pre-publish checklist.

Vale runs the pattern-based rules in CI (`.github/workflows/vale.yml`

- `vale-styles/shariq/`). Lint locally:

```sh
brew install vale
vale src/content/writing
```

Vale warnings/suggestions don't block CI; only errors do (today, only
broken frontmatter). The drafting agent in Plan B is expected to run
Vale before opening its PR.

## AI drafting agent (Plan B)

Three GHA workflows under `.github/workflows/agent-*.yml` run on cron:

- **agent-discover** â€” Sunday 03:00 UTC: scans Notion ideas, recent commits, GH trending, HN, and YT performance; proposes up to 3 blog + 3 YT candidates as Notion rows.
- **agent-draft** â€” Daily 02:00 UTC: picks up Content rows with `Stage=Ready`. Blog rows â†’ MDX file + an AI-generated cover (non-fatal â€” a cover failure never aborts the draft) â†’ PR. YT rows â†’ script blocks written to the Notion row body.
- **agent-promote** â€” Daily 04:00 UTC: for published YT rows with `Cross-post Targets` includes `blog`, mints a linked blog derivative row in `Stage=Proposed`.

Source of truth for content: the restructured Notion CMS DB (see `docs/superpowers/specs/2026-06-02-plan-b-ai-drafting-agent-design.md`).

Local invocation (for prompt iteration without burning CI):

```sh
npm run agent:discover
npm run agent:draft
npm run agent:promote
```

Each reads `.env.local` (see `.env.local.example`).

Style contracts:

- Blog: `docs/EDITORIAL.md`
- YouTube: `docs/SHORTS-STYLE.md`

## Code review on PRs

Two independent AI reviews can run on a PR:

- **`gpt-5.4` via Copilot CLI** â€” a GitHub Action (`review` check + a
  `## ðŸ¤– AI code review` sticky comment). Re-runs automatically on every
  push. Always watch it (and address findings) before calling a PR ready.
- **Built-in GitHub Copilot reviewer** â€” triggered manually:

  ```sh
  gh pr edit <PR#> --add-reviewer copilot-pull-request-reviewer
  ```

  Gotchas: (1) the reviewer slug must be exactly
  `copilot-pull-request-reviewer` â€” not `Copilot`/`copilot`/`github-copilot`
  (its comments post under the author name **Copilot**, but the handle
  differs); a wrong slug errors or silently no-ops. (2) It reviews **once
  per request** and does NOT auto-re-review on new pushes â€” to get fresh
  eyes on later commits, remove then re-add:

  ```sh
  gh pr edit <PR#> --remove-reviewer copilot-pull-request-reviewer
  gh pr edit <PR#> --add-reviewer copilot-pull-request-reviewer
  ```
